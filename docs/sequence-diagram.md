# Sequence Diagram — KYC Onboarding (Happy Path)

**Flow:** Create customer → Submit KYC → Calculate risk score  
**Evidence:** Integration test `tests/test_integration.py`, E2E `tests/e2e/test_platform_e2e.py`

---

## Full Pipeline Sequence

```mermaid
sequenceDiagram
    autonumber
    participant CLI as Node CLI / Client
    participant API as FastAPI
    participant CS as CustomerService
    participant KS as KycService
    participant RS as RiskScoreService
    participant CR as CustomerRepository
    participant KR as KycRepository
    participant DR as DocumentRepository
    participant Pan as PanVerifier
    participant Bank as BankVerifier
    participant DB as Database

    CLI->>API: POST /customers
    API->>CS: create_customer
    CS->>CR: create
    CR->>DB: INSERT customers
    API-->>CLI: 201 {id, status: pending}

    CLI->>API: POST /kyc {customer_id, pan, account, ifsc}
    API->>KS: submit_kyc
    KS->>CR: get_by_id
    KS->>KR: create_submission
    KR->>DB: INSERT kyc_submissions
    KS->>Pan: verify(pan)
    KS->>KR: save_pan_record
    KR->>DB: INSERT pan_records
    KS->>Bank: verify(account, ifsc)
    KS->>KR: save_bank_record
    KR->>DB: INSERT bank_records
    KS->>KR: update status verified
    KS->>CR: update_status kyc_verified
    CR->>DB: UPDATE customers
    API-->>CLI: 201 {status: verified}

    CLI->>API: POST /risk-score {customer_id}
    API->>RS: calculate
    RS->>CR: get_by_id
    RS->>KR: get_latest_by_customer
    RS->>DR: save_risk_assessment
    DR->>DB: INSERT risk_assessments
    API-->>CLI: 200 {score, band, factors}
```
